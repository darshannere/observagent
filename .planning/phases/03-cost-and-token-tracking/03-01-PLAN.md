  <name>Task 2: Create lib/costEngine.js — pricing table, cost formula, usage extraction, context fill</name>
  <files>lib/costEngine.js</files>
  <action>
    Create `lib/costEngine.js` as a pure computation module — no side effects, no imports from other project files (only Node.js built-ins if needed).

    **Export the following:**

    1. `PRICING` — configurable map keyed by exact Claude Code model ID string:
    ```javascript
    export const PRICING = {

      'claude-opus-4-1':            { input: 15.00, output: 75.00, cacheRead: 1.50,  cacheWrite5m: 18.75, cacheWrite1h: 30.00 },
      'claude-sonnet-4-20250514':   { input: 3.00,  output: 15.00, cacheRead: 0.30,  cacheWrite5m: 3.75,  cacheWrite1h: 6.00  },
      'claude-opus-4-20250514':     { input: 15.00, output: 75.00, cacheRead: 1.50,  cacheWrite5m: 18.75, cacheWrite1h: 30.00 },
      'claude-haiku-3-5':           { input: 0.80,  output:  4.00, cacheRead: 0.08,  cacheWrite5m: 1.00,  cacheWrite1h: 1.60  },
    };
    ```
    All rates are USD per million tokens.

    2. `CONTEXT_WINDOWS` — configurable map per model (200K default for all current models):
    ```javascript
    export const CONTEXT_WINDOWS = {
      'claude-sonnet-4-6': 200_000,
      'claude-opus-4-6':   200_000,
      'claude-haiku-4-5':  200_000,
      'claude-sonnet-4-5': 200_000,
      'claude-opus-4-5':   200_000,
      'claude-opus-4-1':   200_000,
      'claude-sonnet-4-20250514': 200_000,
      'claude-opus-4-20250514':   200_000,
      'claude-haiku-3-5':  200_000,
      '_default':          200_000,
    };
    ```

    3. `extractUsageRecords(jsonlRecords)` — filters raw JSONL records and returns usage objects:
    - Input: array of raw parsed JSONL objects
    - Filter to only `type === 'assistant'` records with `message.usage` present
    - CRITICAL dedup rule: skip any record where `msg.stop_reason === null || msg.stop_reason === undefined` (these are streaming-start duplicates — counting them doubles all token counts)
    - For cache writes, prefer nested `msg.usage.cache_creation?.ephemeral_5m_input_tokens` — fall back to top-level `msg.usage.cache_creation_input_tokens ?? 0` (treat all as 5m if nested absent)
    - Return array of objects with shape:
      `{ messageId, model, sessionId, timestamp, inputTokens, outputTokens, cacheReadTokens, cacheWrite5m, cacheWrite1h }`

    4. `computeCost(record)` — returns total cost in USD for one usage record:
    - Lookup `PRICING[record.model]`, fallback to `PRICING['claude-sonnet-4-6']` for unknown models
    - Formula: `(inputTokens * rates.input + outputTokens * rates.output + cacheReadTokens * rates.cacheRead + cacheWrite5m * rates.cacheWrite5m + cacheWrite1h * rates.cacheWrite1h) / 1_000_000`

    5. `getContextFillPercent(model, lastUsage)` — returns integer 0-100:
    - `lastUsage` is the usage object from the most recent final record
    - `totalInput = lastUsage.inputTokens + lastUsage.cacheReadTokens + lastUsage.cacheWrite5m + lastUsage.cacheWrite1h`
    - `contextWindow = CONTEXT_WINDOWS[model] ?? CONTEXT_WINDOWS['_default']`
    - Return `Math.min(100, Math.round((totalInput / contextWindow) * 100))`

    6. `aggregateSessionCost(usageRecords)` — sums all records for a session:
    - Returns `{ model, inputTokens, outputTokens, cacheReadTokens, cacheWrite5m, cacheWrite1h, totalCostUsd, lastEventTs, contextFillPct }`
    - model = from last record (most recent model used)
    - totalCostUsd = sum of `computeCost(r)` for each record
    - lastEventTs = timestamp from last record (highest timestamp)
    - contextFillPct = `getContextFillPercent(model, lastRecord)` using last record's tokens

<task type="auto">
  <name>Task 3: Create lib/jsonlWatcher.js — JSONL discovery, file watching, SQLite upsert, SSE broadcast</name>
  <files>lib/jsonlWatcher.js</files>
  <action>
    Create `lib/jsonlWatcher.js`. This module contains the full JSONL data pipeline: auto-discover session files, watch for changes, re-parse, upsert to SQLite, broadcast via SSE.

    **Imports:** Node.js built-ins only (`node:fs`, `node:fs/promises`, `node:path`, `node:os`, `node:readline`) plus `./costEngine.js` and `./sseClients.js`.

    **Constants:**
    ```javascript
    const PROJECTS_DIR = join(homedir(), '.claude', 'projects');
    // Never hardcode /Users/... — use os.homedir()
    ```

    **Key implementation rules:**
    1. `parseJsonlFile(filePath)` — use `readline.createInterface` with `createReadStream`. For each line: trim, skip empty, try JSON.parse, skip malformed. Returns array of raw objects.

    2. `debounceMap` (module-scope Map) — debounce re-parses at 300ms per file. Rapid JSONL writes during active sessions would thrash CPU without this.

    3. `processFile(filePath, db)` — the core function:
       - Parse JSONL with parseJsonlFile
       - Call `extractUsageRecords(records)` from costEngine
       - If no usage records, return early (skip empty files)
       - Call `aggregateSessionCost(usageRecords)` from costEngine
       - Extract `sessionId` from the filename (basename without `.jsonl`)
       - Upsert to SQLite `session_cost` table using INSERT OR REPLACE:
         ```javascript
         const upsert = db.prepare(`
           INSERT INTO session_cost
             (session_id, model, input_tokens, output_tokens, cache_read_tokens,
              cache_write_5m, cache_write_1h, total_cost_usd, last_event_ts, updated_at)
           VALUES
             (@session_id, @model, @input_tokens, @output_tokens, @cache_read_tokens,
              @cache_write_5m, @cache_write_1h, @total_cost_usd, @last_event_ts, @updated_at)
           ON CONFLICT(session_id) DO UPDATE SET
             model             = excluded.model,
             input_tokens      = excluded.input_tokens,
             output_tokens     = excluded.output_tokens,
             cache_read_tokens = excluded.cache_read_tokens,
             cache_write_5m    = excluded.cache_write_5m,
             cache_write_1h    = excluded.cache_write_1h,
             total_cost_usd    = excluded.total_cost_usd,
             last_event_ts     = excluded.last_event_ts,
             updated_at        = excluded.updated_at
         `);
         ```
         Note: Create this prepared statement once at module scope (passed db), not per-call. Since db is passed at startup, create it lazily on first call or accept db in startJsonlWatcher and close over it.
       - Broadcast SSE event:
         ```javascript
         broadcast({
           type: 'cost_update',
           sessionId,
           cost: agg.totalCostUsd,
           tokens: {
             input:      agg.inputTokens,
             output:     agg.outputTokens,
             cacheRead:  agg.cacheReadTokens,
             cacheWrite: agg.cacheWrite5m + agg.cacheWrite1h,
           },
           contextFillPct: agg.contextFillPct,
           model: agg.model,
           ts: Date.now(),
         });
         ```

    4. `watchFile(filePath, fileWatchers, db)` — sets up individual file watch:
       - Guard: if `fileWatchers.has(filePath)` return
       - `fs.watch(filePath, () => debounceReparse(filePath, fp => processFile(fp, db)))`
       - On watcher error: delete from fileWatchers, close watcher

    5. `startJsonlWatcher(db)` — exported function:
       - Check if PROJECTS_DIR exists using `stat()`, if not log `[cost] ~/.claude/projects/ not found — cost tracking inactive` and return (implements SETUP-02 gracefully)
       - Read all subdirectories in PROJECTS_DIR
       - For each subdirectory: list files, for each `.jsonl` file: call `processFile(fullPath, db)` then `watchFile(fullPath, fileWatchers, db)`
       - Set up recursive directory watch on PROJECTS_DIR:
         ```javascript
         watch(PROJECTS_DIR, { recursive: true }, (event, filename) => {
           if (!filename?.endsWith('.jsonl')) return;
           const fullPath = join(PROJECTS_DIR, filename);
           if (!fileWatchers.has(fullPath)) {
             setTimeout(async () => {
               await processFile(fullPath, db).catch(() => {});
               watchFile(fullPath, fileWatchers, db);
             }, 200); // 200ms delay: new file may not be readable at rename-event time
           }
         });
         ```
       - Log: `[cost] JSONL watcher started — watching ${PROJECTS_DIR}`

    All async errors in processFile and watchFile must be caught silently (cost tracking should never crash the server).
  </action>
