# Plan 10-01: Backend & Database Changes

## Requirements
- AGNT-06: Agent detail panel shows the initial task description / prompt the agent was given when spawned
- AGNT-09: Agent detail panel shows input + output token counts per API call for that agent

## Tasks

### Task 1: Add database schema columns
- **File**: `db/schema.js`
- **Action**: Add three new columns using `addColumnIfNotExists`:
  1. `initial_prompt TEXT` to `agent_nodes` table
  2. `cache_read_tokens INTEGER NOT NULL DEFAULT 0` to `api_calls` table
  3. `cache_write_tokens INTEGER NOT NULL DEFAULT 0` to `api_calls` table
- **Verification**: Run `PRAGMA table_info(agent_nodes)` and `PRAGMA table_info(api_calls)` to confirm columns exist

### Task 2: Update relay.py to capture initial prompt
- **File**: `relay.py`
- **Action**: Modify the agent spawn detection to capture the initial prompt when a Task tool creates a new subagent. Extract from `tool_input.description` or first user message.
- **Verification**: New agents should have their initial_prompt populated in the database

### Task 3: Update relay.py to capture cache tokens
- **File**: `relay.py`
- **Action**: Modify the API call event handling to extract `cache_read_tokens` and `cache_write_tokens` from the API response and send to backend
- **Verification**: New API call events include cache token values

### Task 4: Create GET /api/agents/:id/detail endpoint
- **File**: `routes/api.js`
- **Action**: Add new endpoint that returns:
  - Agent metadata (including initial_prompt)
  - Tool call history for this agent (timestamp, tool_name, duration_ms, exit_status, tool_summary)
  - Token breakdown per API call (timestamp_ms, input_tokens, output_tokens, cache_read_tokens, cache_write_tokens)
- **Verification**: curl localhost:3847/api/agents/{id}/detail returns JSON with all fields
