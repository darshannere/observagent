  <name>Task 2: Fix ingest.js to read exit_status from request body</name>
  <files>routes/ingest.js</files>
  <action>
Update routes/ingest.js to read exit_status from the request body instead of hardcoding null.

Change line 30 from:
```javascript
exit_status:  null,
```

To:
```javascript
exit_status:  raw.exit_status ?? null,
```

The ?? null (nullish coalescing) ensures:
- relay.py sends exit_status: 0 (success) — stored as 0 (falsy but valid; || would wrongly coerce to null)
- relay.py sends exit_status: 1 (failure) — stored as 1
- relay.py omits exit_status (PreToolUse) — stored as null

IMPORTANT: Do NOT use raw.exit_status || null — that coerces 0 (success) to null, breaking the isError check in the dashboard. Must use ?? operator.

No other changes to ingest.js are needed. The pairing logic, broadcast ordering, and write queue are all correct per VERIFICATION.md key link checks.
