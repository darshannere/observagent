    Write frontend/src/pages/HistoryPage.tsx as a self-contained React page component. No Zustand store needed — this page is standalone (history is not live-updated via SSE).

    State (all local useState):
    - sessions: SessionSummary[] (loaded from /api/sessions)
    - loading: boolean
    - error: string | null
    - expandedId: string | null (which session row is expanded)

    Data loading: useEffect on mount, fetch /api/sessions → set sessions.

    Project grouping: derive from sessions using useMemo:
    ```
    const grouped = useMemo(() => {
      const map = new Map<string, SessionSummary[]>()
      for (const s of sessions) {
        const proj = s.project_path.split('/').pop() || s.project_path
        if (!map.has(proj)) map.set(proj, [])
        map.get(proj)!.push(s)
      }
      // Sort each group by start_time desc
      for (const [, arr] of map) arr.sort((a, b) => b.start_time - a.start_time)
      return map
    }, [sessions])
    ```

    Layout structure:
    - Page header: "Session History" title + "Live Dashboard →" link to /live
    - Loading/error state handling
    - For each project group: a shadcn Card with project name as header, session rows inside

    Session row content (compact, not a card — similar to tool log rows):
    - Session ID: `s.session_id.slice(-8)` in monospace
    - Date: `formatTs(s.start_time)` + duration if end_time present: `formatDuration(s.end_time - s.start_time)`
    - Cost: `formatCost(s.total_cost_usd)`
    - Tokens: `formatTokens(s.total_tokens)`
    - Model: truncated model name (last segment after '/')
    - Error count: only shown if > 0, in red text
    - Active badge: green "● active" badge if is_active
    - Export buttons: "JSONL" and "CSV" — see export handler below
    - Replay button: "Replay →" link to `/live?replay=${s.session_id}`
    - Click on the row (excluding buttons): toggle expandedId to show/hide event summary

    Export handler:
    ```typescript
    async function exportSession(sessionId: string, format: 'jsonl' | 'csv') {
      const res = await fetch(`/api/sessions/${sessionId}/export?format=${format}`)
      const text = await res.text()
      const blob = new Blob([text], { type: format === 'csv' ? 'text/csv' : 'application/x-ndjson' })
      const url = URL.createObjectURL(blob)
      const a = document.createElement('a')
      a.href = url
      a.download = `session-${sessionId.slice(-8)}.${format}`
      a.click()
      URL.revokeObjectURL(url)
    }
    ```
    This is the same pattern as the vanilla JS triggerDownload function.

    Expanded session row: when expandedId matches, show a sub-row with additional details:
    - Full session_id
    - Project path (full)
    - Start time and end time (if present)
    - event_count

    Navigation: use Link from react-router for "Live Dashboard →" and "Replay →" links. Import `{ Link }` from 'react-router'.

    Styling: dark theme consistent with LiveDashboard — use shadcn Card component for project groups, compact rows with monospace text for session entries (same row style as ToolLogRow: px-2 py-1 font-mono text-xs).
