Per CONTEXT.md locked decision: "A toggle in the observagent_config table enables forwarding the full raw tool_input JSON instead of specific fields. Default is off." This is API-only in Phase 8 — no UI control.

In db/schema.js initDb(), after the existing observagent_config CREATE TABLE block (line ~65), add a seed INSERT that runs only if the key is not already set:

  console.log('[db] full_tool_input_enabled config seeded (default: off)');

This follows the same INSERT OR IGNORE pattern used for budget_threshold_usd and ctx_fill_threshold_pct in the rest of the codebase.

In routes/ingest.js, after the event object is fully built (after tool_summary field is added in Task 1), add a conditional block that checks the config and stores raw tool_input for debugging only when the toggle is enabled:

  // full_tool_input_enabled toggle — API-controlled, default off
  // When enabled, raw tool_input JSON is logged to console for debugging (not stored in events table)
  const fullInputEnabled = db.prepare(`SELECT value FROM observagent_config WHERE key = 'full_tool_input_enabled'`).get()?.value === '1';
  if (fullInputEnabled && raw.tool_input) {
    console.log('[ingest] full_tool_input:', JSON.stringify(raw.tool_input));
  }

Implementation note: The db prepare call inside the route handler is acceptable for Phase 8 (low-frequency ingest events). The raw tool_input is NOT stored in the events table — only logged to console when enabled. This keeps the events schema unchanged beyond the tool_summary column from Task 1. The toggle is activated by a direct DB update (e.g., UPDATE observagent_config SET value='1' WHERE key='full_tool_input_enabled') or via the existing /api/settings routes if applicable.
