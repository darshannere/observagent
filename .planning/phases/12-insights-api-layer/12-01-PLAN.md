**Prepared statements (define once at registration):**

`stmtCostDaily`:
```sql
SELECT
  date(last_event_ts) AS day,
  SUM(total_cost_usd) AS cost_usd
FROM session_cost
WHERE agent_id = ''
  AND last_event_ts >= date('now', '-6 days')
GROUP BY date(last_event_ts)
ORDER BY day ASC
```
Returns up to 7 rows, one per calendar day. Days with no activity are absent (frontend fills gaps).

`stmtCostByAgent`:
```sql
SELECT
  COALESCE(NULLIF(an.agent_type, ''), 'solo') AS agent_type,
  SUM(sc.total_cost_usd) AS cost_usd
FROM session_cost sc
LEFT JOIN agent_nodes an ON an.agent_id = sc.agent_id
WHERE sc.agent_id != ''
GROUP BY agent_type
ORDER BY cost_usd DESC
```
Returns cost totals grouped by agent_type. For solo sessions (agent_id = '' in session_cost, no agent_nodes row), they are excluded from this view since they have no agent_type — the cost-daily endpoint covers total daily spend. If no agent sub-costs exist, returns empty array (not an error).

**Route handlers:**

`GET /api/insights/cost-daily`:
- Call `stmtCostDaily.all()`
- Return array of `{ day: "2026-03-10", cost_usd: 0.0412 }` objects

`GET /api/insights/cost-by-agent`:
- Accept optional query param `session_id` (ignored for now — reserved for Phase 13 filtering)
- Call `stmtCostByAgent.all()`
- Return array of `{ agent_type: "gsd-executor", cost_usd: 0.0231 }` objects
