  <name>Task 1: Write Zustand store (useObservStore.ts) with full state and all action reducers</name>
  <files>frontend/src/store/useObservStore.ts</files>
  <action>
    Create frontend/src/store/ directory and write useObservStore.ts.

    The store must hold:
    - agents: Map&lt;string, Agent&gt; (keyed by agentId)
    - sessions: Map&lt;string, Session&gt; (keyed by parentSessionId — the root session groupings)
    - events: ToolEvent[] (chronological, all sessions)
    - activeSessionFilter: string | null (URL-reflected filter — null means show all)
    - sessionCosts: CostStateEntry[] (from /api/cost hydration)
    - todayCost: number
    - costModels: ModelCost[]
    - config: Config | null
    - health: HealthState | null
    - sseConnected: boolean

    Actions:
    - addAgent(data: Omit&lt;Agent, 'cost' | 'tokens'&gt;): Registers agent; creates parent Session if not exists; appends agentId to session.children. Uses new Map() for immutability.
    - updateAgentState(agentId, state, ts): Updates agent state and lastActivityTs. Uses new Map().
    - updateAgentCost(agentId, cost, tokens): Updates agent cost/tokens; rolls up total cost/tokens for the parent session. Uses new Map().
    - appendEvent(event: ToolEvent): Appends to events array. Uses spread: [...s.events, event].
    - updateEventDuration(tool_call_id: string, duration_ms: number, exit_status: number): Finds the matching PreToolUse event by tool_call_id and updates its duration_ms and exit_status in place — this is the live-update-in-place pattern for PostToolUse. Uses map() to create new array.
    - hydrateEvents(events: ToolEvent[]): Replaces events array entirely (initial load). Also reconstructs inProgressMap for any events without duration.
    - setCostData(sessions: CostStateEntry[], todayTotal: number, models: ModelCost[]): Sets sessionCosts, todayCost, costModels.
    - setConfig(config: Config): Sets config.
    - setSessionFilter(sessionId: string | null): Sets activeSessionFilter.
    - setHealth(h: HealthState): Sets health.
    - setSseConnected(v: boolean): Sets sseConnected.

    CRITICAL: Zustand 5 requires double-parentheses syntax: create&lt;ObservStore&gt;()((set, get) => ...)

    CRITICAL: All Map mutations MUST create new Map instances — `const agents = new Map(state.agents)` — then mutate and return `{ agents }`. Mutating in place breaks Zustand's reference equality check and prevents re-renders.

    The updateAgentCost action must roll up session totals: iterate all children in session.children, sum cost + tokens from agents Map, then store on the Session object. This is identical to the vanilla JS logic in public/index.html lines 1229–1249.

    Export: `export const useObservStore = create&lt;ObservStore&gt;()(...)`

<task type="auto">
  <name>Task 2: Write useSSE hook — single EventSource lifecycle with full message dispatch</name>
  <files>frontend/src/hooks/useSSE.ts</files>
  <action>
    Create frontend/src/hooks/ directory and write useSSE.ts.

    The hook signature: `export function useSSE(isReplay = false): void`

    When isReplay is true, skip opening EventSource entirely (the history page uses this for ?replay= mode — just fetches historical data without live SSE).

    Implementation:
    1. Use `useRef&lt;EventSource | null&gt;(null)` to store the EventSource — NOT useState (state causes re-render loops).
    2. In useEffect with empty deps []:
       - If isReplay, return immediately (no-op).
       - If esRef.current exists, close it first (guards against React StrictMode double-mount).
       - Create `new EventSource('/events')` and assign to esRef.current.
       - Set `es.onopen` → `useObservStore.getState().setSseConnected(true)`.
       - Set `es.onerror` → `useObservStore.getState().setSseConnected(false)`.
       - Set `es.onmessage` with full message dispatch:
         - `type === 'connected'` → return (no-op)
         - `type === 'agent_spawn'` → store.addAgent({ agentId: msg.agentId, parentSessionId: msg.parentSessionId, agentType: msg.agentType || '', state: 'active', lastActivityTs: msg.ts || Date.now() })
         - `type === 'agent_update'` → store.updateAgentState(msg.agentId, msg.state, msg.ts || Date.now())
         - `type === 'cost_update' && msg.agentId` → compute totalTokens from msg.tokens (input + output + cacheRead + cacheWrite), call store.updateAgentCost(msg.agentId, msg.cost, totalTokens)
         - `type === 'health_update'` → store.setHealth({ totalCalls: msg.totalCalls, errorCount: msg.errorCount, errorRate: msg.errorRate, lastEventTs: msg.lastEventTs })
         - `msg.hook_type === 'PreToolUse'` → store.appendEvent(msg as ToolEvent)
         - `msg.hook_type === 'PostToolUse'` → call store.updateEventDuration(msg.tool_call_id, msg.duration_ms, msg.exit_status ?? 0) to patch the in-progress row
       - Return cleanup: `() => { es.close(); esRef.current = null; useObservStore.getState().setSseConnected(false) }`

    IMPORTANT anti-pattern to avoid: Do NOT call `useObservStore()` (with parentheses, the hook) inside useEffect — that subscribes the hook component to the store. Instead use `useObservStore.getState()` (static getter, no subscription) inside the effect.

    Replay note: The LiveDashboard page will pass `isReplay={true}` when `?replay=SESSION_ID` is in the URL. In that case, the page fetches the session's historical events directly via `/api/events?session_id=X` and does not open SSE.
